# AI Game Studio - 实施路线图与开发任务拆分文档

## 1. 总目标
用多份文档驱动 Cursor 和本地后端，把系统从零实现到可演示、可交互、可产出小游戏。

## 2. 实施阶段
### Stage A：基础工程骨架
- 建立前端项目
- 建立后端项目
- 跑通本地开发环境

### Stage B：静态控制台
- 完成办公室基础界面
- 展示 5 个角色
- 展示电脑与队列区
- 暂时使用本地假数据

### Stage C：后端状态系统
- 场景状态接口
- 角色状态接口
- 消息与日志接口
- 任务与阶段接口
- 电脑锁接口

### Stage D：Producer 驱动流程
- 用户可向制作人提交需求
- 制作人可发起澄清
- 制作人可生成阶段计划
- 用户可批准计划

### Stage E：Agent 调用接入
- 接入 Cursor Cloud Agents API
- 连接 5 个角色 Agent
- 支持 webhook 回写

### Stage F：电脑锁与排队
- Developer 申请电脑
- 锁状态可视化
- 执行完成后释放
- 队列推进

### Stage G：小游戏落地
- Designer 产出玩法说明
- Artist 产出 UI 规范
- Developer 实现点击成长小游戏
- QA 完成测试与反馈

### Stage H：演示闭环
- 用户可从需求开始
- 可看到协作过程
- 可批准阶段
- 可最终运行小游戏

## 3. 关键里程碑
- 办公室界面能跑起来
- 后端状态可驱动界面
- 制作人能产出计划
- 电脑锁成立
- Developer 能真改代码
- 小游戏可运行
- QA 闭环成立
